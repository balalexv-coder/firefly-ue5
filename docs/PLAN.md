# План работ — Firefly UE5

Восемь фаз. Каждая фаза самодостаточна: результат можно протестировать и отложить, не блокируя следующую. Порядок рекомендуется, но не обязателен — dialogue server, например, можно начать с фазы 0, параллельно всему остальному.

Лишь примерная оценка времени (вечерами, один человек, с нуля): **~5–7 недель**. Быстрее — если купить готовые ассет-паки (западный город, лошади).

## Фаза 1 — Dialogue server (1–2 дня)

**Цель:** рабочий HTTP-сервис, который на любую реплику игрока и состояние разговора возвращает JSON с репликами персонажей.

**Можно делать параллельно фазе 0**, от UE не зависит.

**Задачи:**
1. Установить Python 3.11+, создать venv в `tools/dialogue_server/`.
2. `pip install -r requirements.txt` (fastapi, uvicorn, anthropic, pydantic).
3. Положить ключ Anthropic в `.env` (шаблон в `.env.example`).
4. Запустить сервер: `uvicorn server:app --reload --port 8765`.
5. Прогнать тестовый клиент: `python test_client.py`.

**Контракт API** детально описан в [DIALOGUE_SYSTEM.md](DIALOGUE_SYSTEM.md).

**Deliverable:** `POST /turn` возвращает валидный JSON с 2–3 репликами + 3 предложенными вариантами ответа для игрока.

## Фаза 2 — Моделирование экипажа (3–5 дней)

**Цель:** 9 MetaHuman-персонажей, узнаваемо напоминающих экипаж «Serenity».

**Задачи:**
1. Открыть **MetaHuman Creator** (в UE5 встроен начиная с 5.5).
2. Создать 9 персонажей по рецептам из [CHARACTERS.md](CHARACTERS.md). Начните с 2-3 ключевых (Mal, Kaylee, Wash) — это даёт рабочую сцену.
3. Собрать каждого в Bridge, скачать в UE (Quality: Highest для героев, Medium для сцены в кают-компании).
4. Назначить каждому голос-ID для TTS (см. [LIP_SYNC.md](LIP_SYNC.md#voices)).
5. Одежда: MetaHuman поставляются с базовыми нарядами; для «западного» стиля — либо купить `Modular Cowboy Outfit` с Marketplace/Fab, либо оставить базовые на первом проходе.

**Риск:** MetaHuman без лицензионного футажа — это *похожий*, не копия. Не заявляйте, что это «те самые актёры».

**Deliverable:** 9 BP_*Crew-актёров, каждый со своим голосом и базовой анимацией idle.

## Фаза 3 — Кают-компания и рассадка (2–3 дня)

**Цель:** помещение со столом и 9 персонажами, сидящими на своих местах; корректные позы.

**Задачи:** подробно в [SEATING_ANIMATION.md](SEATING_ANIMATION.md). Кратко:
1. Собрать помещение из модульных стен/пола/мебели (можно купить `Sci-Fi Interior Kit` или собрать из куба + материалов под дерево/металл).
2. Большой прямоугольный стол — центральный объект, как в сериале.
3. Для каждого стула создать сокет `ChairSeat`.
4. Скачать анимации с Mixamo: `Sitting Idle`, `Sitting Talking (1/2/3)`, `Sitting Typing`.
5. Ретаргетить анимации на MetaHuman-скелет через **IK Retargeter**.
6. Blueprint `BP_SeatActor`: при BeginPlay берёт указанного персонажа и снаппит в сокет, включает Sit Animation.
7. AnimBlueprint `ABP_SeatedCrew`: машина состояний Idle ↔ Talking ↔ Reacting, управляется параметрами, выставляемыми из C++/BP снаружи.

**Deliverable:** все 9 сидят за столом, idle-анимация играет, камера ставит кадр как в сцене из сериала.

## Фаза 4 — Диалоговая система в UE (3–5 дней)

**Цель:** UI + логика, которая ходит в dialogue server, озвучивает реплики, анимирует говорящего и возвращает управление игроку.

**Задачи:** подробно в [DIALOGUE_SYSTEM.md](DIALOGUE_SYSTEM.md#ue-side). Кратко:
1. **UMG-виджет** `WBP_DialogueHUD`:
   - Внизу экрана: плашка с именем говорящего + текст реплики, печать по буквам.
   - Когда очередь игрока: 3 кнопки с вариантами.
2. **C++ Actor Component** `UDialogueClientComponent`:
   - Хранит историю реплик.
   - `RequestTurn()` — POST JSON на `localhost:8765/turn`.
   - Парсит ответ → эмитит события для HUD и `BP_SeatActor`.
3. **TTS + озвучка:**
   - Вариант A (быстро, бесплатно): сабтитры без озвучки.
   - Вариант B (реалистично): dialogue server вызывает TTS (ElevenLabs / OpenAI / локальный Kokoro), кеширует WAV, URL передаёт в UE; UE проигрывает через Sound Cue.
4. **Анимация говорящего:**
   - При старте реплики — AnimMontage из набора "Sitting Talking".
   - Камера-cut на говорящего (см. [SCENES.md](SCENES.md#camera-coverage)).

**Deliverable:** запускаешь уровень — экипаж здоровается, выбираешь реплику, команда отвечает. Цикл из 4–5 раундов заканчивается подходом к атмосфере.

## Фаза 5 — Орбитальный экран (1–2 дня)

**Цель:** большой экран/иллюминатор в кают-компании, на котором видно «Serenity», подлетающую к планете.

**Задачи:**
1. Отдельный sublevel `L_OrbitalScene` — маленькая «сценка в банке»: модель корабля, модель планеты (ваша версия с NASA-текстурой), космический фон.
2. `Scene Capture 2D` camera смотрит на корабль. Render Target → материал экрана в кают-компании.
3. `LevelSequence` двигает корабль: дальняя орбита → приближение → заход в атмосферу.
4. По ходу диалога (каждый раунд) `DialogueClientComponent` продвигает таймлайн на 1 шаг: корабль ближе.
5. Последний раунд диалога = корабль у атмосферы → триггерим переход в синематик посадки.

**Deliverable:** живой экран, который меняется в такт разговору.

## Фаза 6 — Синематик посадки (3–5 дней)

**Цель:** кинематографический шот, как «Serenity» садится на пустыню в стиле сериала.

**Задачи:** детали в [CINEMATIC.md](CINEMATIC.md). Кратко:
1. Новый sublevel `L_Landing` с ландшафтом (пустыня).
2. Rigged-корабль: отдельные пивоты для двух двигателей-гондол, которые поворачиваются из «cruise» (параллельно оси) в «hover» (вниз).
3. Niagara-системы:
   - `NS_EntryHeat` — плазма/жар при входе в атмосферу (ventral, headfront).
   - `NS_EngineExhaust` — outflow из сопел, реагирует на угол.
   - `NS_LandingDust` — большая пыльная буря на touchdown.
4. `LevelSequence` `LS_Landing` — 4 шота:
   - Shot A (0–6s): корабль по дуге входит в атмосферу, камера вровень.
   - Shot B (6–14s): силуэт над дюнами, двигатели поворачиваются.
   - Shot C (14–22s): приближение к земле, пыль клубится.
   - Shot D (22–28s): кадр с земли на упершийся корабль, пыль садится.
5. Переход: Sequencer → Level Blueprint → Open `L_Town` с игроком у трапа.

**Рендер в видео** (для публикации): [RENDER_EXPORT.md](RENDER_EXPORT.md).

**Deliverable:** играбельная цепочка `L_SerenityCabin → L_Landing → L_Town`.

## Фаза 7 — Ковбойский городок (7–10 дней)

**Цель:** компактный, но плотный фронтир-городок. Главная улица, 6–10 зданий, 15–25 NPC, лошади, атмосфера.

**Задачи:** детали в [TOWN_DESIGN.md](TOWN_DESIGN.md). Кратко:
1. **Ассеты.** Сильно упрощает жизнь покупка паков:
   - *Western Town* (Fab/Marketplace) — здания, мостовые, реквизит.
   - *Horse Animset Pro* или аналог — анимации лошади.
   - *Western Props Pack* — мелочи.
   - Альтернатива: моделить в Blender + импортировать через Datasmith / Interchange.
2. **PCG** раскладывает дома вдоль главной улицы с заданными интервалами, заполняет проулки и дворы реквизитом.
3. **Landscape** — пустыня с редкими полевыми травами (у вас уже есть опыт с ландшафтами).
4. **NPC:**
   - 5 архетипов (горожанин, ковбой, торговец, пьяница, шериф).
   - Simple behavior tree: Wander → Idle → Talk → Wander.
   - Schedule: дневной vs вечерний пресет (толпа у салуна вечером).
5. **Лошади:**
   - 2–3 у коновязи (idle, хвост дёргают).
   - 1 всадник едет по улице.
6. **Саунд:**
   - Ambience: ветер, скрип дерева, цикады, далёкий лай собаки.
   - Точечные: пианино из салуна, кузнечный молот, ржание.

**Deliverable:** можно походить по городку, NPC двигаются, лошади дышат.

## Стратегические решения

| Вопрос | Решение | Альтернатива | Когда пересмотреть |
|---|---|---|---|
| Озвучка реплик | TTS через внешний API (ElevenLabs), кеш на диск | Без озвучки, только сабы | Если API дорого → Kokoro/XTTS локально |
| Лип-синк | NVIDIA Audio2Face → ARKit blendshapes | MetaHuman Animator с видео | Если A2F встанет — Oculus LipSync |
| Модели персонажей | MetaHuman Creator | Blender + Character Creator 4 | Для лиц героев достаточно MetaHuman |
| Городок | Платный пак с Fab | PCG + kitbash из Megascans | Если пак дорогой — Megascans бесплатны |
| Лошади | Платный пак (Horse Animset Pro) | Без лошадей на v0.1 | Нужны позже — купить единожды |
| LLM | Claude 4.7 Sonnet/Opus | GPT-4o / локальный Llama | Если Claude API недоступен → OpenAI |

## Риски и как их смягчать

1. **Ассетная война.** Искать "правильный" пак — легко потерять неделю. Решение: купить первые попавшиеся подходящие паки на $60–100 суммарно, не перфекционировать.
2. **Лип-синк качеством расстроит.** Даже плохой липсинк лучше «говорящих мешков». Рекомендую стартовать с MetaHuman Animator на сгенерированных TTS-дорожках.
3. **Синематик в MP4 не рендерится.** Movie Render Queue на Windows не умеет MP4 из коробки — нужен FFmpeg на PNG sequence. Подробно в [RENDER_EXPORT.md](RENDER_EXPORT.md).
4. **Производительность в городке.** 25 NPC + лошади + ландшафт + MetaHuman → FPS просядет. На v0.1 — упрощаем NPC (low-poly, не MetaHuman), MetaHuman только у экипажа.
5. **Копирайт.** Если выкладывать на YouTube — дисклеймер "fan-project, non-commercial".
